## 🎯 Business Goal

**Goal**: 부업을 시작하고 싶은 한국인 누구나, 사기성 광고 없이 신뢰할 수 있는 실질적인 부업 정보를 한 곳에서 탐색하고 즉시 행동에 옮길 수 있는 정보형 애그리게이터 플랫폼의 MVP를 빠르게 실체화한다.

**성공 측정 기준 (Go/No-Go)**:
- **Primary KPI**: 방문자의 **30% 이상**이 상세 페이지를 **2개 이상** 클릭 (Engagement 검증)
- **Secondary**: 5명 이상의 실제 사용자가 "광고·사기 사이트와 다르다"고 인식

---

## ❗ Problem Statement

**우리가 해결하는 문제**:

> 부업/N잡을 시작하려는 사람들이 정보를 검색할 때, 대부분의 결과가 사기성 광고·강의 유도·허위 수익 과장으로 오염되어 있어, **신뢰할 수 있는 실질적인 부업 정보를 한 곳에서 찾을 수 없다.**

**결과적 손실**:
- 정보 탐색에 수시간을 낭비
- 신뢰할 수 없는 정보로 인해 잘못된 부업 선택
- "어디서 시작해야 할지 모름"으로 인한 실행력 저하 → 아무것도 시작하지 못함

**우리가 해결하지 않는 문제** (Out of Scope):
- 부업 매칭/채용 플랫폼 (잡코리아, 크몽 역할은 아님)
- 실시간 수익 추적 또는 커뮤니티 기능
- 유료 강의 또는 컨설팅 연결

## 🗺️ User Journey Map

| 단계 | 트리거 | 행동 | 감정 | 결과 |
|------|--------|------|------|------|
| **Awareness** (인식) | 고정 지출 증가, "부업으로 월 500만원" 유튜브 광고 노출 | 구글/네이버에 "부업 추천", "N잡 방법", "직장인 부업" 검색 | 호기심 + 기대감 | 수십 개 결과 중 대부분이 광고성 블로그, 강의 결제 유도 페이지 |
| **Consideration** (고려) | 클릭할 때마다 유료 강의 랜딩 페이지로 연결 | 여러 사이트 비교, 유튜브 영상 시청, 오픈채팅 가입 | ⚠️ **피로감 + 강한 불신** | "믿을 수 있는 정보를 한 곳에서 보고 싶다"는 니즈 극대화 |
| **Adoption** (채택) | 우리 플랫폼 발견 (SEO 또는 SNS 유입) | 홈 피드에서 카드형 부업 목록 스크롤 탐색 | 신선함 + 신뢰감 ("광고 없네?", "깔끔하다") | 카드 2~3개 클릭, 상세 페이지 진입 |
| **Implementation** (실행) | 상세 페이지에서 난이도/수익/비용 정보 확인 | '시작 가이드' 섹션 읽고 첫 단계 실행 또는 북마크 | 자신감 + 실행 의지 | 실제 부업 시작 or 재방문 |

> **최대 고통 지점 (🔴 개입 포인트)**: **Consideration 단계** — 정보를 찾을수록 광고·사기에 지쳐 포기하는 순간. 우리 플랫폼이 이 단계의 탈출구가 되어야 함.

## 🏗️ Data Architecture Decisions

> SPRINT-001에서 확정된 데이터 구조 방향성. 개발 단계 전 Scope Freeze.

### 데이터 규모 (MVP)

| 항목 | 결정 |
|------|------|
| 초기 항목 수 | **30~50개** 고퀄리티 부업 항목 |
| 세분화 깊이 | **대분류 단위** (스마트스토어 = 1개 항목) |
| 서브 정보 처리 | 상세 페이지 '시작 가이드' 섹션 내 텍스트로 세부 방향 안내 |

### JSON 스키마 설계 원칙

- **파일명**: `public/data/side-hustles.json`
- **분류 방식**: **태그(Tag) 기반 다중 분류** (하나의 부업이 여러 카테고리 동시 소속 가능)
- **큐레이션**: 초기에는 운영자 수동 큐레이션. Boolean 필드로 관리

```json
{
  "id": "couang-partners",
  "title": "쿠팡파트너스",
  "slug": "coupang-partners",
  "summary": "쿠팡 상품 링크를 공유해 구매 발생 시 수수료를 받는 제휴마케팅",
  "difficulty": "beginner",
  "expectedMonthlyIncome": { "min": 5, "max": 100, "unit": "만원" },
  "initialCost": { "amount": 0, "note": "무료 시작 가능" },
  "timeToFirstIncome": "1~4주",
  "tags": ["제휴마케팅", "온라인마케팅", "무자본"],
  "isTrending": false,
  "isPopular": true,
  "overview": "...",
  "startGuide": ["계정 신청", "링크 생성", "채널 공유 방법"],
  "relatedLinks": [],
  "lastUpdated": "2026-06-17"
}
```

### 확장성 고려 (Post-MVP)

- `isTrending` / `isPopular` 필드는 추후 AI 크롤러나 외부 API(Google Trends 등)로 자동 갱신 가능하도록 설계
- `lastUpdated` 필드로 데이터 신선도 관리
- 스키마 변경 없이 필드 추가 가능한 구조 유지

### 기술 아키텍처 (확정)

| 항목 | 결정 |
|------|------|
| 프레임워크 | **Next.js SSG** (또는 Vite + React) — JAMstack |
| 데이터 저장 | 정적 JSON 파일 (`side-hustles.json`) |
| 배포 | Vercel 또는 GitHub Pages |
| 서버 비용 | $0 목표 (정적 파일 서빙) |
| 광고 준비 | Google AdSense 배너 레이아웃 영역 미리 확보 |

## ❓ Sprint Questions (5개)

> Day 5(Validate)에 실제 사용자 테스트로 답해야 할 핵심 질문들. 이 답변에 따라 SPRINT-002 방향이 결정됨.

1. **신뢰도**: 홈 피드의 카드 형태 부업 정보가 첫 화면에서 충분히 신뢰감 있고 깔끔하게 느껴지는가? 사기성 사이트와 시각적으로 명확히 차별화되는가?

2. **DB 완성도 인식**: 30~50개의 부업 항목이 "여기엔 뭔가 많다"는 충분한 데이터베이스 느낌을 주기에 충분한가, 아니면 빈약하게 느껴지는가?

3. **상세 페이지 실용성**: [개요 / 난이도 / 예상 수익 / 초기 비용 / 시작 가이드] 구성이 유저가 실제 행동을 취하게 만드는 실질적인 정보로 느껴지는가?

4. **핵심 KPI**: 방문자의 **30% 이상**이 상세 페이지를 **2개 이상** 클릭하는가?

5. **탐색 UX**: 태그 기반 카테고리 필터가 유저의 자연스러운 탐색 흐름을 지원하는가, 아니면 오히려 사용을 방해하는가?
